# 测试与答辩讲解

## 一、测试目标

本项目的测试重点不是覆盖所有细节，而是验证系统运行、配置读取、前端状态和核心业务链路是否稳定。

## 二、主要测试文件

当前项目中较关键的测试包括：

- `util/tests_configread.py`
  验证配置读取与字符集兼容逻辑。
- `util/tests_startup_scripts.py`
  验证本地启动脚本和样例环境变量是否齐全。
- `templates/front/admin/tests/security-cleanup.test.mjs`
  验证登录态和会话清理逻辑。
- `templates/front/admin/tests/forecast-state.test.mjs`
  验证预测工作台相关状态行为。
- `templates/front/admin/tests/forecast-workspace-config.test.mjs`
  验证预测页配置收口逻辑。
- `main/tests.py`
  验证后端业务接口、车型知识采集、训练命令和部分实验链路。

## 三、系统验证重点

系统验证主要覆盖以下方面：

- 配置与环境变量读取是否正常。
- 启动脚本是否能够完成基础自举。
- 前端登录态、会话和工作台状态是否正确。
- 后端业务接口、训练命令和实验链路是否可运行。

## 四、演示路径

系统演示可以按以下顺序展开：

1. 首页与总览页。
2. 行车日志列表页。
3. 预测工作台。
4. NASA 实验页。

该路径能够覆盖业务管理、预测分析和实验展示三类主要功能。
