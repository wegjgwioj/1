# 项目讲解稿实施计划

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**目标：** 将仓库中的旧英文讲解文档替换为一套全中文、贴合论文主线且适合学习本项目的分文件讲解稿，并完成一次仅包含文档改动的 Git 提交。

**架构：** 先删除旧英文文档，再按“项目导读 → 系统架构 → 后端 → 前端 → 预测实验 → 部署运行 → 测试答辩 → 论文写作 → 相关文献”的顺序新建中文文档，最后在 `README.md` 中补充推荐阅读顺序，并执行文档范围提交。

**技术栈：** Markdown、Git、PowerShell、Django 项目目录、Vue 3 前端目录、Windows 本地部署说明。

运行：`git status --short`
预期：识别出本次需要提交的是文档相关文件，同时注意 `main/tests.py` 属于未完成代码改动，不纳入这次提交。
