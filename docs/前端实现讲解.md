# 前端实现讲解

## 一、前端的职责

前端负责将业务管理、预测分析和实验展示组织为统一的管理端界面。其主要任务包括：

- 展示行车日志业务数据。
- 承接预测工作台输入与结果展示。
- 展示 NASA 实验相关指标和图像结果。
- 统一处理路由、请求和登录态。

## 二、前端入口结构

前端位于 `templates/front/admin/`，核心入口包括：

- `src/main.js`
- `src/App.vue`
- `src/router/`
- `src/utils/`
- `src/views/`

这些目录分别承担应用启动、路由组织、公共工具和页面实现职责。

## 三、页面结构

### 1. 首页与总览

首页与总览页用于汇总系统核心能力，作为行车日志管理、预测分析和实验展示的统一入口。

### 2. 行车日志列表页

列表页承载行车日志业务主线，主要功能包括：

- 查询与筛选
- 详情查看
- 收藏与评论
- 车型知识采集

### 3. 预测工作台

预测工作台用于接收场景参数、加载模型输出并展示预测结果，是前端中承接智能分析功能的核心页面。

### 4. NASA 实验页

NASA 页面用于展示电池寿命实验相关内容，通常包括：

- 数据准备结果
- SOH / RUL 指标
- 图像结果
- 实验说明文字

## 四、前端公共工具

### 1. `src/utils/http.js`

`http.js` 是前后端交互的统一入口，用于收口请求发送、错误处理和鉴权相关逻辑。

### 2. `src/utils/base.js`

`base.js` 负责读取和拼接后端基础地址，是前端环境变量配置的重要入口。

### 3. `src/utils/adminSession.js`

`adminSession.js` 负责登录态与会话清理逻辑，用于减少旧标签页残留、退出不彻底和状态错乱等问题。

## 五、前端结构特征

当前前端实现具有以下特点：

- 页面结构围绕业务、预测和实验三条主线展开。
- 请求入口和基础地址由公共工具统一管理。
- 登录态和会话清理由专门工具收口。
- 预测工作台和 NASA 页面承担结果展示职责。
