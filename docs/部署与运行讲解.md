# 部署与运行讲解

## 一、主部署方式

本项目以 Windows 本地自举作为主要部署方式，目标是在常见本地环境下通过脚本完成依赖安装、数据库初始化和前后端启动。

## 二、部署相关文件

关键文件主要包括：

- `README.md`
- `bin/bootstrap_local.ps1`
- `bin/start_project.ps1`
- `bin/start_backend.ps1`
- `bin/bootstrap_local_db.py`
- `.env.example`
- `config.ini`

这些文件共同构成了本项目的本地部署方案。

## 三、首次启动流程

首次部署流程如下：

1. 安装 Python 3.11、Node.js 18+、MySQL，以及可选的 Redis。
2. 执行 `bin/bootstrap_local.ps1`。
3. 由脚本自动完成虚拟环境创建、依赖安装、环境变量文件生成、数据库初始化以及前后端启动。

## 四、环境变量处理

项目的环境变量采用统一收口方式处理：

- 后端优先读取 `.env`，不足时回退到 `config.ini`。
- 前端通过 `VITE_API_BASE_URL` 等变量统一接口地址。
- `README.md` 对常用变量提供说明。

该方式用于减少数据库地址、端口和接口地址在不同环境中的错位问题。

## 五、Docker 文件的定位

仓库中保留了 `docker-compose.yml` 和相关 Docker 文件，但当前仅作为补充方案存在，不是默认运行路径。

## 六、运行目标

当前部署方案的目标包括：

- 保证项目在本地环境中可完成初始化。
- 保证数据库和环境变量按脚本约定完成配置。
- 保证前后端可通过统一脚本启动。
