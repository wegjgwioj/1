# 预测与实验讲解

## 一、这一部分是整个项目最容易讲乱的地方

本项目同时包含机器学习主模型、深度学习对比实验和 NASA 电池 SOH/RUL 实验。很多人在介绍时会把三者混成一句“我做了深度学习预测”，结果反而让老师追问时很难自圆其说。

更清晰的讲法是把它们拆成三条职责不同的链路：

- 机器学习主链路：服务业务系统默认预测输出。
- 深度学习对比链路：服务方法对比与补充实验。
- NASA 实验链路：服务论文研究深度与真实寿命预测验证。

## 二、机器学习主链路为什么是主系统默认输出

### 1. 数据特点决定了方法选择

业务主数据是结构化行车日志，它天然更适合使用特征工程较明确、解释性较强的机器学习方法来建模。这样做有三个好处：

- 模型更稳定。
- 指标更容易解释。
- 页面展示更容易和字段含义对应起来。

因此项目把机器学习链路作为主系统默认输出，是一个兼顾可用性、可解释性和答辩稳定性的选择。

### 2. 关键文件

机器学习主链路主要位于：

- `main/ml/feature_engineering.py`
- `main/ml/trainers.py`
- `main/ml/evaluation.py`
- `main/ml/predictors.py`

对应命令主要是：

- `python manage.py train_ml_models`
- `python manage.py evaluate_ml_models`

如果你要读源码，建议先看特征构造，再看训练器，最后看预测器。因为只有先理解输入特征是怎么来的，后面的模型输出才有意义。

## 三、深度学习对比链路承担什么作用

### 1. 它不是来取代主模型的

这一点一定要讲清楚。深度学习链路的价值主要体现在：

- 为论文提供方法对比。
- 为序列建模提供补充思路。
- 说明项目对更复杂模型方案做过探索。

它并不是为了强行证明“深度学习一定更好”。在当前数据规模和标签构造方式下，机器学习主模型更适合作为稳定输出，这本身就是一个真实且合理的工程结论。

### 2. 关键文件

深度学习链路主要位于：

- `main/dl/sequence_dataset.py`
- `main/dl/lstm_model.py`
- `main/management/commands/prepare_sequence_data.py`
- `main/management/commands/train_dl_model.py`
- `main/management/commands/compare_models.py`

系统还考虑到了答辩机器环境差异，因此当 `torch` 不可用时，会自动回退到轻量神经网络近似实现。这一点非常适合写进论文，因为它体现了“研究探索”与“工程落地”之间的折中处理。

## 四、NASA 实验为什么必须单独拿出来讲

### 1. 它补的是研究深度，不是业务功能

业务系统中的寿命预测和 NASA 数据集上的 SOH / RUL 预测，在概念上不是一回事：

- 业务预测更偏系统功能与辅助分析。
- NASA 实验更偏真实电池退化规律建模。

项目把 NASA 实验单独保留出来，是为了让论文中出现真正有研究属性的实验内容，从而避免整个课题只剩“后台系统 + 普通预测页”的薄弱印象。

### 2. 关键文件

NASA 链路主要位于：

- `main/battery_life/nasa_dataset.py`
- `main/battery_life/trainers.py`
- `main/management/commands/prepare_nasa_battery_dataset.py`
- `main/management/commands/train_nasa_battery_models.py`

这一部分的训练产物通常会输出到 `artifacts/battery_life/`，便于在页面与论文中同时引用。

## 五、三条链路之间的正确关系

这三条链路的关系可以这样概括：

- 机器学习主链路是主系统可用性的核心。
- 深度学习对比链路是方法丰富性的体现。
- NASA 实验链路是研究深度与论文说服力的来源。

如果答辩时老师问“为什么既有机器学习又有深度学习，还要再做 NASA 实验”，你可以回答：

“因为三者解决的问题不同。机器学习负责系统默认预测输出，深度学习负责对比实验，NASA 数据集实验负责在公开真实电池老化场景下验证 SOH 和 RUL 预测能力，它们共同构成了系统实现和论文研究的完整闭环。”

## 六、从学习角度应该怎么读这一部分

建议按下面顺序：

1. 先读 `main/ml/feature_engineering.py`
2. 再读 `main/ml/trainers.py`
3. 然后看 `main/ml/predictors.py`
4. 接着看 `main/dl/sequence_dataset.py`
5. 再看 `main/management/commands/train_dl_model.py`
6. 最后读 `main/battery_life/nasa_dataset.py` 和 `main/battery_life/trainers.py`

这样能先把业务预测主线看清楚，再去理解对比实验和 NASA 实验为什么存在。

## 七、论文中这一章怎么写更稳

论文里的写法建议分成三小节：

- 一节写业务预测主链路。
- 一节写深度学习对比实验。
- 一节写 NASA 电池寿命实验。

每一节都讲清楚下面四件事：

- 数据来源是什么。
- 采用了什么建模思路。
- 产物存在哪里。
- 它在整个项目中承担什么作用。

这样写出来的论文结构会很稳，不会把所有实验和预测混成一大团。
