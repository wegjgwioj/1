# 项目导读

## 一、这不是一个普通的后台模板

第一次看这个仓库时，最容易产生的误解是：它只是一个基于 Django 和 Vue 的管理系统模板，外加一些电动汽车字段。实际上，这个项目已经被整理成一套更接近毕业设计交付的系统，它同时承担了三层任务：

- 完成一个可运行的电动汽车行车日志业务系统。
- 完成一条能耗与寿命代理预测的智能分析链路。
- 通过 NASA 公开电池老化数据集补充 SOH 和 RUL 实验，增强论文研究深度。

所以，学习这个项目时不能只盯着页面，也不能只盯着算法。更合适的方式是把它理解成“业务系统 + 智能分析 + 研究实验”三条线并行推进的一体化项目。

## 二、建议的阅读顺序

如果你是第一次接触这个项目，建议按下面顺序阅读：

1. 先读《系统架构讲解》，理解项目整体怎么分层、数据从哪里来、结果最终流向哪里。
2. 再读《后端实现讲解》，看 Django 这一侧如何组织接口、模型和管理命令。
3. 接着读《前端实现讲解》，理解页面入口、状态管理、接口调用和展示逻辑。
4. 然后读《预测与实验讲解》，重点理解机器学习主链路、深度学习对比链路和 NASA 实验的边界。
5. 再读《部署与运行讲解》，理解为什么这个项目强调 Windows 本地自举与一键启动。
6. 最后读《测试与答辩讲解》和《论文写作讲解》，把项目实现与答辩表达、论文结构对应起来。

如果你是老师或答辩评审，通常读完《系统架构讲解》《预测与实验讲解》《论文写作讲解》三篇，就已经能把项目主线看明白。

## 三、阅读源码时应该先看哪里

### 1. 后端入口

后端核心目录集中在以下位置：

- `dj2/`：全局配置、数据库、缓存、中间件装配。
- `main/`：业务模型、接口文件、训练逻辑、管理命令。
- `util/`：配置读取、认证、公共工具。
- `xmiddleware/`：请求预处理与鉴权链路。

建议的后端阅读入口是：

1. `dj2/settings.py`
2. `main/urls.py`
3. `xmiddleware/xauth.py`
4. `main/Drivinglog_v.py`
5. `main/Drivinglogforecast_v.py`
6. `main/ml/`、`main/dl/`、`main/battery_life/`

### 2. 前端入口

前端管理端位于 `templates/front/admin/`，推荐阅读顺序是：

1. `src/main.js`
2. `src/router/`
3. `src/utils/http.js`
4. `src/utils/adminSession.js`
5. `src/views/home/`
6. `src/views/list/`
7. `src/views/forecast-workbench/`
8. `src/views/board/`

这样读的好处是先把应用入口、页面跳转和请求方式搞清楚，再进入具体页面，不容易迷路。

## 四、项目的三条主线

### 1. 业务主线

业务主线围绕“行车日志管理与展示”展开，包含日志列表、筛选查询、收藏、评论、管理员管理、车型知识采集等能力。这一条线解决的是系统“能不能用”的问题。

### 2. 智能分析主线

智能分析主线围绕“能耗与寿命代理预测”展开。这里的重点不是追求最复杂的模型，而是让系统可以在业务场景中稳定给出预测结果。这一条线解决的是系统“有没有分析能力”的问题。

### 3. 研究实验主线

研究实验主线围绕 NASA 电池老化公开数据展开，用于完成 SOH 和 RUL 建模与结果展示。这一部分并不直接替代业务预测，而是作为论文的研究补强。这一条线解决的是项目“有没有研究深度”的问题。

## 五、学习这个项目时要特别注意的边界

### 1. 业务预测不等于 NASA 实验

业务预测来自行车日志结构化数据，更偏向系统功能演示与辅助分析。NASA 实验来自公开电池老化数据，更偏向研究验证。两者的数据来源、标签定义和应用目标不同，不能混在一起讲。

### 2. 机器学习主模型不等于深度学习对比模型

这个项目默认以机器学习结果作为主展示链路，因为它在当前数据条件下更稳定、可解释。深度学习部分的价值主要在于方法对比和实验补充，而不是强行替代主系统输出。

### 3. 可演示版本不等于最终可扩展形态

论文中保留了 Hadoop、Hive 等扩展性思路，但当前答辩交付版本的重点是“本地能启动、功能能演示、结果能复现”。学习项目时要先抓住已落地的部分，不要一上来就被扩展设计带偏。

## 六、阅读完成后的预期收获

读完整套讲解稿后，你应该能够回答下面几个问题：

- 这个项目到底解决了什么问题。
- 为什么要同时保留业务系统、预测链路和 NASA 实验。
- 后端和前端的主要入口分别在哪里。
- 训练产物为什么会放在 `artifacts/`。
- 论文中每一章应该怎样和代码对应起来。

如果这些问题都能答出来，说明你已经不只是“看过代码”，而是真的进入了这个项目的设计语境。
