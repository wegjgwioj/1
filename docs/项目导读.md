# 项目导读

## 一、项目定位

本项目是一套围绕电动汽车行车日志数据构建的综合分析系统，采用 `Django + Vue 3` 技术栈实现。系统同时包含业务管理、智能预测和电池寿命实验三部分内容，目标是将行车日志数据管理、分析和实验验证整合到同一套平台中。

项目的核心组成包括：

- 行车日志业务管理系统。
- 能耗与寿命代理预测链路。
- 基于 NASA 公开电池老化数据集的 SOH 与 RUL 实验链路。

## 二、主要目录

项目的核心目录如下：

- `dj2/`：全局配置、数据库、缓存和中间件装配。
- `main/`：业务模型、接口文件、训练逻辑和管理命令。
- `util/`：配置读取、认证和公共工具。
- `xmiddleware/`：请求预处理与鉴权链路。
- `templates/front/admin/`：Vue 3 前端管理端。
- `artifacts/`：模型文件、指标报告和实验产物。
- `docs/`：项目讲解、部署说明和相关材料。

## 三、项目主线

### 1. 业务主线

业务主线围绕行车日志管理与展示展开，覆盖日志列表、筛选查询、收藏、评论、管理员管理和车型知识采集等能力。

### 2. 预测主线

预测主线围绕能耗与寿命代理预测展开，负责将结构化行车日志数据转换为可用于业务展示的预测结果。

### 3. 实验主线

实验主线围绕 NASA 电池老化公开数据展开，用于完成 SOH 与 RUL 建模和实验结果输出。

## 四、边界说明

### 1. 业务预测与 NASA 实验不是同一链路

业务预测基于行车日志结构化数据，面向系统分析与展示；NASA 实验基于公开电池老化数据，面向电池退化建模验证。两者的数据来源、标签定义和目标不同。

### 2. 机器学习主模型与深度学习对比模型承担不同角色

项目默认以机器学习结果作为主展示链路，深度学习链路主要承担方法对比和实验补充角色，并不直接替代主系统输出。

### 3. 当前实现优先保证可运行和可复现

项目保留了部分扩展性设想，但当前版本的重点是完成本地可启动、功能可运行、实验结果可复现的交付形态。
