# 论文写作讲解

## 一、论文主线应该怎样统一

这篇论文最忌讳写成“功能堆砌”。正确的主线应该围绕下面这句话展开：

“针对电动汽车行车日志数据利用率不高、分析维度单一、智能应用不足的问题，设计并实现一套集数据管理、可视化分析、智能预测与实验验证于一体的电动汽车行车日志分析系统。”

有了这条主线之后，论文中的系统设计、系统实现、模型实验和部署测试就都能被纳入同一个叙事框架，而不会显得彼此割裂。

## 二、论文章节如何和项目对应

### 1. 绪论

绪论主要承担三个任务：

- 说明选题背景与研究意义。
- 说明国内外相关研究现状。
- 说明本文工作的主要内容与创新点。

在这一章里，你要把“业务系统”和“研究实验”两个维度都提到，但不要一上来展开太细。重点是先建立问题意识和研究动机。

### 2. 相关理论与关键技术

这一章对应项目中的技术基础，主要可以写：

- Django 后端框架。
- Vue 3 前端框架。
- MySQL 与可选 Redis。
- 机器学习与深度学习建模基础。
- NASA 数据集相关实验背景。

如果论文还保留 Hadoop、Hive 相关内容，建议写成“面向海量日志分析的扩展性设计基础”，不要写得像当前版本完全依赖它们运行。

### 3. 系统分析与总体设计

这一章最适合对应《系统架构讲解》的内容，建议重点写：

- 功能需求。
- 非功能需求。
- 系统总体架构图。
- 主要模块划分。
- 数据流与训练产物流。

这里是老师判断你是否具备系统设计能力的重要章节。

### 4. 系统实现

这一章对应《后端实现讲解》《前端实现讲解》，建议重点写：

- 后端模块组织方式。
- 主要业务接口与模型设计。
- 前端页面结构与交互方式。
- 车型知识采集、收藏评论、预测工作台等关键功能实现。

这里不要逐个函数堆代码，而要围绕模块职责与关键实现思路来写。

### 5. 模型设计与实验验证

这一章对应《预测与实验讲解》，最好拆成三部分：

- 机器学习主预测链路。
- 深度学习对比实验。
- NASA 电池 SOH / RUL 实验。

每一部分都要明确说明数据来源、方法思路、实验结果和在项目中的定位。

### 6. 系统测试与部署

这一章可以对应《部署与运行讲解》《测试与答辩讲解》，重点写：

- 环境配置。
- 一键启动与本地自举。
- 配置读取和启动脚本验证。
- 前端状态与后端接口测试。
- 系统运行截图与结果截图。

这一章很能体现你的项目完成度。

### 7. 结论与展望

结论不是重复前文，而是总结：

- 系统实现了什么。
- 实验验证了什么。
- 当前版本还有哪些不足。
- 后续可以如何扩展。

## 三、论文里必须讲清楚的边界

### 1. 业务预测与 NASA 实验的边界

这两个概念一定要分开讲。业务预测来自行车日志，面向系统功能展示；NASA 实验来自公开电池老化数据，面向 SOH / RUL 研究验证。二者不能互相替代，也不能写成同一套结论。

### 2. 机器学习主链路与深度学习对比实验的边界

机器学习是主系统默认输出，深度学习是对比实验和方法补充。这样写既符合实际，也避免出现“为什么最终页面不用深度学习结果”的追问。

### 3. 当前交付版本与未来扩展方向的边界

论文中可以保留扩展性设计，但要明确说明当前交付版本优先强调可运行、可演示和可复现实验，而不是夸大尚未完全落地的部分。

## 四、论文中的图片应该怎么安排

建议至少放以下几类图片：

- 系统总体架构图
- 首页截图
- 行车日志列表截图
- 预测工作台截图
- NASA 实验页截图
- NASA 预测图或衰减曲线图

仓库中已有可直接使用的截图资源，位于：

这些图片的作用不只是美化页面，而是让论文中的“系统设计、系统实现、实验结果”三部分都有直观证据。

## 五、如何降低论文中的模板味和空话感

建议注意以下几点：

- 少用“显著提升、极大增强、完美实现”这类空泛词。
- 多写真实取舍，例如“为保证答辩现场稳定性，本系统默认以机器学习主链路作为预测展示输出”。
- 适当写出路径、命令、目录和产物位置，让文字更像真实项目总结。
- 在实验结论中保持克制，不要强行夸大深度学习结果。

## 六、论文最终应该给老师留下什么印象

最理想的印象不是“这个人会调模板”，而是：

“他围绕一个明确问题完成了系统设计、前后端实现、预测与实验链路搭建，并进一步完成了测试验证、部署收口和论文表达整理，因此这个项目具备完整的毕设交付质量。”

如果你的论文能够稳定传达出这一点，整体完成度就已经很高了。
